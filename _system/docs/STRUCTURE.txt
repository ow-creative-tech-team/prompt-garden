KEY CONCEPTS
================================================================================

📋 YAML FRONTMATTER (Required at top of every file)
   → Metadata: id, title, type, category, tags, version, dates, etc.
   → See templates for complete structure

🔗 MODULAR SYSTEM
   → Style + Scene + Subject = Complete composition
   → Link prompts using 'related_prompts' field in YAML

📝 NAMING CONVENTIONS
   Folders:  lowercase-kebab-case
   Files:    _style.md, scene-*.md, subject-*.md, or descriptive-name.md

🌿 GIT WORKFLOW
   1. Pull latest (git pull origin main)
   2. Create branch (git checkout -b feature/name)
   3. Create/edit prompts
   4. Commit (git commit -m "feat(category): description")
   5. Push (git push origin feature/name)
   6. Create PR on GitHub

================================================================================
QUICK COMMANDS
================================================================================

Start new prompt:
  cd text-to-image/styles/photography
  mkdir my-style
  cp ../../../_system/templates/style-template.md my-style/_style.md
  code my-style/_style.md

Daily workflow:
  git pull origin main
  git checkout -b feature/my-prompt
  [create/edit files]
  git add . && git commit -m "feat(type): description"
  git push origin feature/my-prompt

Check status:
  git status
  git diff
